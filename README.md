## 一句话

**DiVo Agent Platform** 是企业级 AI Agent 运行时的治理基座。基于 **DiVo 自研的 Agent Harness Engineering 技术**，通过 **垂直化的环境沙箱隔离与策略驱动架构**，让每台 AI Agent 既拥有充分自主性，又被严格锁在你定义的数据、模型、供应链安全边界之内。

---

## 我们解决什么问题

企业想用 AI Agent，但面临三对矛盾：

| 矛盾 | 现状 |
|---|---|
| **能力 vs. 合规** | Agent 越聪明，越需要自主访问外部资源；企业越需要合规，越需要锁死边界 |
| **个性化 vs. 一致** | 不同业务部门定制自己的 Agent 体验，但 IT 部门必须保证每一台都能被统一纳管 |
| **快速上线 vs. 自主可控** | SaaS 产品上手快但数据出域；自己开发又没团队、没工期 |

**DiVo Agent Platform** 的答案是：**环境即治理单元**。

把 Agent 关进 DiVo Harness 托管的垂直化沙箱：
- LLM 端点 —— **只放行你指定的一家**
- 工具与插件 —— **白名单准入、全链路可审计**
- 网络出口 —— **走你指定的中继网关**
- 运行日志 —— **全链路可回查**

## 为什么是 DiVo

### 环境沙箱隔离
DiVo Agent Harness 为每个 Agent 任务级实例提供独立的垂直化沙箱 —— 进程、存储、网络三层完全隔离。一个 Agent 被污染，不影响任何其他实例、不污染宿主机。

### 策略驱动治理
IT 在平台统一定义"允许什么、禁止什么"，DiVo Harness 运行时自动对所有 Agent 工作负载强制生效，不需要每台 Agent 单独配。IT 改一条策略，全局即刻对齐。

### LLM 路由 · 全链路可管控
- 平台内所有 Agent 的 LLM 调用强制走 **企业自有的 LLM 中继网关**
- 一条策略定义，覆盖全部 Agent 实例，不多也不少
- **国产大模型全面适配**：DiVo Harness 已在 LLM 管理层深度适配国内主流国有模型厂商，包括 **百度文心一言、阿里通义千问、腾讯混元、华为盘古、智谱 GLM 系列、DeepSeek 系列** 等。企业混部自有模型与国有模型时，DiVo 帮您做统一路由、统一审计、统一核算，不需要各团队自己对接 API

### 企业身份 · 接入现有账号体系
- **OIDC / SAML / CAS**：开箱即对接 Keycloak、CAS、企业 AD 等
- 组/角色继承：Agent 运行身份 = 你的员工账号，天然继承你在企业中的权限
- SSO 单次登录，覆盖所有工作空间

### Skills & MCP · 开放架构，可控边界
- **Skills 仓库**：IT 审核通过的 Skill 才能在平台上架，业务自助启用
- **MCP Servers**：平台作为 MCP Client 的托管环境，单向暴露内部 MCP Service，禁止工作空间私建出站 MCP 通道
- **插件签名上架**：企业私有市场替代公有市场，IT 控制入口

### 合规审计 · 全链路可追溯
- 每次 Agent 活动的 API 调用、模型请求、工具调用**全部结构化落库**
- 用户提示词指纹（SHA-256 hash，不存原文）支持**事后归因**
- 策略违规立即触发告警、自动隔离

## 五大典型用户场景

### 场景 1️⃣ 垂直 IT 服务企业
> **"我们有十几个交付团队，每个团队都用 Cursor / Copilot，既想用又怕数据泄"**

- 每个交付项目 = 一个独立的 DiVo Harness Agent 沙箱
- Agent 只能调用企业网关中的 LLM，**收件箱代码绝不出境**
- 项目结束，沙箱一键销毁，不留下任何残留

### 场景 2️⃣ 国企/大型集团
> **"全集团要铺 AI，但集团信息部要能控、能查、能合规"**

- 统一纳管：所有二级单位、子公司的 Agent 都在同一平台内跑
- 按法人实体切分沙箱隔离边界，数据不出法人边界
- 集团审计处直读审计库，不需要求下级单位上报

### 场景 3️⃣ 政府/事业单位
> **"我们要国产化、要物理隔离、要等保合规"**

- 完全物理断网部署支持
- 接入国产 LLM 网关（**对接你已有的内部 LLM 基础设施**）
- 日志格式直接对接等保三级/四级审计要求

### 场景 4️⃣ 科研/高校
> **"实验室要跑大量 Agent 做科研，希望 IT 部门代建基座，自己专注 Prompt 和 Skills"**

- IT 统一沙箱规格，科研团队自助领取
- 资源限额：每团队最多同时跑 X 个活跃沙箱
- 科研成果（Skills、配置）可被 IT 捕获成标准资产

### 场景 5️⃣ 中小微企业 & 独立开发者
> **"我想在自己服务器上跑 Agent，但不希望环境失控、不想学架构"**

- **一体机镜像**：一个可执行文件 + 一条命令就起平台
- 免运维：自携带数据库、自动更新、夜间快照备份
- 免费起步，成长后升级企业版解锁 SSO、审计和 HA
