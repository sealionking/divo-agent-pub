# 工作空间模板：IT 的治理抓手

> **一句话：模板定义边界，IT 定义模板，Agent 跑在边界里。**

---

## 什么是模板

模板是一份 **可版本化、可复用、可审计** 的工作空间规范。它声明：

- ✅ 工作空间可以**使用哪些** Skill / 工具 / 扩展（白名单制）
- ✅ LLM 请求只能走**哪一条路由**（不允许 Agent 私自选择别家网关）
- ✅ 出站网络**策略**（默认断网 + 白名单放行）
- ✅ **可用模型清单**（预置国内主流企业大模型，不私自引入未知来源）
- ✅ 工作空间跑完**留下什么**（日志保留多久、快照策略）

模板一旦发布，业务用户自助领取，不需要 IT 一台一台配。

---

## 模板例子（抽象描述）

一个典型的"研发 Agent"模板：

```
这个模板定义：
  ● 名称：标准研发空间
  ● LLM 路由：全走 newapi.wangjueju.cn 网关
  ● 可用模型：
      - Claude Sonnet 4（企业网关代理）
      - 百度文心 4.0
      - 通义千问 Max
      - DeepSeek V3
  ● 代码访问：被限制在企业内部 Git 仅读
  ● 允许读入的 MCP：企业私有 API（IP 白名单）
  ● 不允许安装任何未经 IT 签名的扩展
  ● 日志保留：90 天
```

用户看到的就是一个下拉菜单 + "创建"按钮，IT 不需要写文档解释每一条限制。

---

## IT 怎么建模板

1. **创建**：声明模板基线（模型清单、白名单工具、出站网关）
2. **本地测试**：IT 内部跑一遍典型场景确认没有超配
3. **发布**：上线 Repository，业务部门可领取
4. **迭代**：下一版本修改后灰度上线，平滑切换

每一次变更入流水，审计可追溯。

---

## 模板 vs. 用户自定义

| 维度 | 模板定义（IT 控） | 用户在模板内自定义 |
|---|---|---|
| LLM 网关 | ✗ 不能改 | — |
| 可用模型列表 | ✗ 不能改 | — |
| 出站网络白名单 | ✗ 不能改 | — |
| Skill / 扩展 | ✓ 只能启用模板允许的子集 | 选装 / 卸载 |
| Prompt / 聊天文件 | — | ✓ 自由编辑 |
| API Key（BYOK） | — | ✓ 用户自己配（选项） |

模板就是硬性边界的公式：**边界之内，用户自主；边界之外，IT 一口回绝**。
